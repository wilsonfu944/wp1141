## 🔧 設置 OAuth（可選但推薦）

### 最簡單：GitHub OAuth（5分鐘）

1. **訪問 GitHub**
   ```
   https://github.com/settings/developers
   ```

2. **創建 OAuth App**
   - 點擊 "New OAuth App"
   - Application name: `X Social Platform`
   - Homepage URL: `http://localhost:3000`
   - Authorization callback URL: `http://localhost:3000/api/auth/callback/github`

3. **獲取憑證**
   - 複製 Client ID
   - 生成並複製 Client Secret

4. **更新 .env 文件**
   編輯 `/Users/joy.lin/Desktop/wp1141/hw5/.env`:
   ```env
   GITHUB_ID="你的-github-client-id"
   GITHUB_SECRET="你的-github-client-secret"
   ```

5. **重啟服務器**
   - 在終端按 `Ctrl + C` 停止服務器
   - 運行 `npm run dev` 重新啟動

### Google OAuth 設置
詳見：`ENV_SETUP.md` 文件

### Facebook OAuth 設置
詳見：`ENV_SETUP.md` 文件

## ⚠️ 注意事項

### OAuth 憑證
目前 `.env` 文件中的 OAuth 憑證都是占位符：
```env
GOOGLE_CLIENT_ID="your-google-client-id"
GITHUB_ID="your-github-id"
FACEBOOK_CLIENT_ID="your-facebook-client-id"
```

**這些需要替換為真實的憑證才能登入。**

### Pusher（即時功能）
即時更新功能需要 Pusher 憑證。目前使用占位符：
```env
NEXT_PUBLIC_PUSHER_KEY="your-pusher-key"
```

沒有 Pusher，其他功能仍可正常使用，只是按讚/評論不會即時同步。

要設置 Pusher：
1. 訪問 https://pusher.com/
2. 註冊免費帳號
3. 創建 Channels 應用
4. 複製憑證到 `.env`

## 🆘 常見問題

### Q: 無法訪問 localhost:3000
**A**: 檢查開發服務器是否正在運行：
```bash
lsof -ti:3000
```
如果沒有輸出，重新啟動：
```bash
cd /Users/joy.lin/Desktop/wp1141/hw5
npm run dev
```

### Q: OAuth 登入失敗
**A**: 確認：
1. OAuth 憑證已正確設置在 `.env`
2. 回調 URL 在 OAuth 提供商設置中正確
3. 重啟了開發服務器

### Q: 資料庫連接錯誤
**A**: 確認 PostgreSQL 正在運行：
```bash
/Library/PostgreSQL/17/bin/pg_ctl status -D /Library/PostgreSQL/17/data/
```
